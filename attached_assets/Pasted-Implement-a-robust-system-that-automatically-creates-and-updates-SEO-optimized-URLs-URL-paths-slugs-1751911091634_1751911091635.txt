Implement a robust system that automatically creates and updates SEO-optimized URLs (URL paths/slugs) based on the current title or header of each exam page or resource.
Example:
If the exam title is “PMP Exam Help 1”, the URL should be:
https://brainliest.com/pmp/pmp-exam-help-1
If the title changes, the URL should update to reflect the new title, using an SEO-friendly format.

Instructions:
Slug Generation Logic

Create a utility function that converts any exam title or header into a URL-safe, SEO-friendly slug:

Lowercase all letters

Replace spaces, underscores, or special characters with hyphens

Remove/replace non-alphanumeric characters (except hyphens)

Collapse multiple hyphens into one

Trim leading/trailing hyphens

Example in JavaScript:

js
Copy
Edit
function slugify(title) {
  return title
    .toLowerCase()
    .replace(/[^a-z0-9\s-]/g, '')    // Remove invalid chars
    .replace(/\s+/g, '-')            // Replace spaces with hyphens
    .replace(/-+/g, '-')             // Collapse multiple hyphens
    .replace(/^-+|-+$/g, '');        // Trim hyphens
}
// "PMP Exam Help 1" => "pmp-exam-help-1"
Dynamic URL Assignment

When creating or updating an exam/title, auto-generate the slug and assign it as the URL path.

Store the slug in the database with the exam record.

Update the URL each time the title changes, ensuring uniqueness (append an increment or ID if needed).

Example:

js

// On exam creation/update:
const basePath = '/pmp/';
const examSlug = slugify(examTitle);
const urlPath = basePath + examSlug;
// Save urlPath in database with the exam
Uniqueness & Canonical URLs

If a generated slug already exists for another exam, automatically append a unique suffix or exam ID (e.g., pmp-exam-help-1-2).

Always set canonical meta tags to the latest URL.

Auto-Redirect Old URLs

If an exam title changes and thus the slug changes, automatically redirect the old URL to the new one (301 redirect) to preserve SEO value and prevent broken links.

Front-End Integration

Ensure all links, breadcrumbs, and navigation automatically use the current, generated slug.

On page render, dynamically fetch the correct content based on the URL slug.

SEO Meta Tags

Update meta titles, descriptions, and canonical links dynamically to match the new slug and title for each page.

Admin Panel UI (optional)

In the admin/editor interface, display the auto-generated URL beneath the title input and update it in real-time as the title is edited.

Optionally, allow manual override with validation for uniqueness and SEO safety.

QA and Testing

Test with a wide range of exam titles (including special characters, numbers, different cases).

Confirm redirects, canonical tags, and all links are correctly updated on slug/title change.

Ensure there are no duplicate or broken URLs.
